# 설계 문서 (Design)

이 디렉터리(`docs/ko/design/`)는 **QMTL 개선을 위해 작성하는 설계 문서**를 둡니다.  
목표는 “아이디어를 남기는 것”이 아니라, **아키텍처/구현에 반영될 수 있도록** 계약·경계·용어·결정사항을 정리하는 것입니다.

## `design/` vs `design/icebox/`

- `docs/ko/design/`:
  - 설계에 반영(또는 반영 예정)하기 위한 **활성 설계 문서**를 둡니다.
  - 이 폴더의 문서는 작업/리뷰/구현을 위한 기준점 역할을 합니다(단, 최종 SSOT는 코드와 `docs/ko/architecture/`입니다).
- `docs/ko/design/icebox/`:
  - **현재 작업 대상이 아닌 참고용 문서**를 둡니다.
  - 스파이크 노트, 초안/스케치, 과거 제안, 아직 채택되지 않은 아이디어 등을 보관합니다.
  - 문서 내용이 오래되었거나 일부만 유효할 수 있으므로, 인용 시 “icebox(참고용)”임을 명시하고 필요한 내용은 아키텍처/코드로 승격해 반영하세요.

## 라이프사이클 (구현 완료 후 정리)

디자인 문서는 **구현을 위한 작업 문서**입니다. 구현이 완료되면:

1) 결정사항/계약/불변조건을 `docs/ko/architecture/`(필요 시 `docs/ko/operations/`)와 코드/테스트에 **정식으로 녹여 SSOT를 단일화**합니다.  
2) 해당 디자인 문서는 **아카이빙 또는 삭제**합니다.
   - 역사적 맥락/의사결정 기록이 유용하면 `docs/ko/archive/`로 이동하고 `status: archived`로 표시합니다.
   - 더 이상 참조 가치가 없거나 내용이 전부 정식 문서로 흡수되면 삭제합니다.

## 링크/인용 가이드

- 다른 문서에서 `design/icebox/*`를 링크할 때는, 링크 텍스트 또는 문장에 **“icebox(참고용, 현재 작업 대상 아님)”**임을 함께 적습니다.
- 의사결정/불변조건/운영 계약처럼 규범이 필요한 내용은 `docs/ko/architecture/` 또는 코드/테스트로 옮겨 SSOT를 단일화합니다.

## 국제화(i18n)

Korean(`docs/ko/...`)이 기준 문서이며, English(`docs/en/...`)는 번역본입니다.  
자세한 규칙은 [문서 국제화 가이드](../guides/docs_internationalization.md)를 참고하세요.
